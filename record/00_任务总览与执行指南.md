## 四、人员分配建议

| 角色 | 建议分配 | 任务 |
|------|---------|------|
| 人员A（熟悉市场数据） | Phase-1a + Phase-1d | 内容审校需要对三份草稿的数据逐一比对；冲突决策需要商业判断力 |
| 人员B（熟悉Word排版） | Phase-1b | 需要在Word中逐页检查，对docx格式有经验 |
| 人员C（能上网核实链接） | Phase-1c | 需要逐条打开URL验证，耗时但技术门槛低 |
| 人员D（会Python/图表） | Phase-2 | 需要用matplotlib或plotly制作图表 |
| 整合人（项目负责人或指定） | Phase-3 | 汇总所有成果，生成终版。需对全局有把握 |

如果人手不足（例如只有2人），建议分配为：
- 人员1：Phase-1a → Phase-1d → Phase-3（核心内容线）
- 人员2：Phase-1b + Phase-1c → Phase-2（格式与图表线）

## 五、质量检查点（Gate）

### Gate 1：Phase-1 全部完成后

在启动 Phase-2 之前，项目负责人需确认：
- [ ] Phase-1a 审校意见已输出，无"高严重"遗漏项未被标记
- [ ] Phase-1d 冲突决策已有推荐方案（不要求决策层已拍板，但分析要到位）
- [ ] Phase-1b 排版问题已记录（无需此时修复）
- [ ] Phase-1c 参考文献核实已完成，失效链接已标记

### Gate 2：Phase-2 完成后

在启动 Phase-3 之前，项目负责人需确认：
- [ ] 6–8张图表已生成，数据与报告正文一致
- [ ] 图表使用的数据来源已在清单中注明
- [ ] 图表配色和中文标签可读性良好

### Gate 3：Phase-3 完成后（最终交付）

- [ ] 终版docx在Word中打开正常，目录可更新
- [ ] 所有图表在文档中正常显示
- [ ] 参考文献中无失效链接（或已标注）
- [ ] 精简版执行摘要与终版数据一致
- [ ] record文件夹下所有过程文档齐全
