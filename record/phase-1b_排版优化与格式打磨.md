## 一、项目背景（请先阅读）

我们有一份已生成的合并版调研报告（docx格式），主题是"中国饮品品牌进入加拿大安大略省市场的可行性研究"。该文件由 Node.js 脚本（使用 docx-js 库）自动生成，并经过 Phase-0 深化研究阶段更新，新增了多个章节（2.5、4.1.1、8.0、8.4等）和扩充的表格内容。

你的任务是：**在 Microsoft Word 或 WPS 中打开这份文件，检查排版效果，记录所有格式问题。** 特别注意Phase-0新增章节的格式是否与原有章节保持一致。

## 三、检查清单

请在 Word/WPS 中打开文件，逐项检查以下内容，并在每项后标注"通过 ✅"或"问题 ❌ + 具体描述"：

### （一）整体页面设置

- [ ] 纸张大小是否为 A4（210mm × 297mm）
- [ ] 页边距是否约为上下左右各1英寸（2.54cm）
- [ ] 正文字体是否为 Arial 或系统中文字体（如宋体/微软雅黑），12号
- [ ] 行间距是否为 1.5 倍

### （二）标题层级

- [ ] 一级标题（如"一、项目背景与研究框架"）：是否为16号加粗，前后有适当间距
- [ ] 二级标题（如"1.1 项目背景"）：是否为14号加粗
- [ ] 三级标题（如有）：是否为12号加粗
- [ ] 各级标题的编号是否连续无跳跃

### （三）目录（Table of Contents）

- [ ] 是否存在目录页
- [ ] 右键点击目录 → "更新域" → "更新整个目录"，目录是否能正常更新
- [ ] 目录中的章节标题是否与正文一致
- [ ] 目录中是否包含所有一级和二级标题
- [ ] 目录页码是否大致正确

### （四）表格

报告中应包含以下主要表格，请逐一检查：

| 表格编号 | 内容 | 检查要点 |
|---------|------|---------|
| 表1 | 三档投资模型冲突对照 | 5列（投资档位、草稿一、草稿二、草稿三、建议区间），3行数据 |
| 表2 | 投资明细拆解（标准店） | 2列（项目、金额区间），约6行 |
| 表3 | 月度运营成本结构 | 3列（费用项、月度区间、占比目标），约9行 |
| 表4 | 开店流程时间线 | 3列（阶段、活动、时间），约7行 |
| 表5 | 三种经营情景 | 7列，3行数据 |
| 表6 | 盈亏平衡分析 | 5列，3行数据 |
| 表7 | 核心KPI | 3列（指标、目标值、说明），约11行 |
| 表8 | 三阶段扩张路径对照 | 4列，3行 |
| 表9 | 数据冲突对照表（附录A） | 5列，8行 |

对每张表格检查：
- [ ] 表格是否完整显示（未被截断或溢出页面）
- [ ] 列宽是否合理（文字未被挤压或出现大面积空白）
- [ ] 表头是否有底色区分（浅灰或浅蓝）
- [ ] 边框线是否清晰可见
- [ ] 中文内容是否正常显示（无乱码）
- [ ] 金额中的 `$` 符号和数字是否清晰

### （五）分页与页面布局

- [ ] 每个一级标题（"一、""二、"……"十二、"）是否从新页开始
- [ ] 是否存在多余的空白页
- [ ] 表格是否被不合理地跨页拆分（一张表格被切到两页）
- [ ] 参考文献部分是否从新页开始

### （六）页眉与页脚

- [ ] 是否有页眉（内容应为报告标题或简写）
- [ ] 是否有页脚页码
- [ ] 页码是否连续

### （七）参考文献格式（APA第七版）

- [ ] 参考文献是否按作者/机构字母顺序排列
- [ ] 是否采用悬挂缩进格式（首行顶格，续行缩进约1.27cm/0.5英寸）
- [ ] URL 是否以超链接形式呈现（蓝色可点击）
- [ ] 格式是否统一：`作者. (年份). 标题. URL`

### （八）特殊标注

- [ ] 文中的"[冲突提示]"标注是否醒目（应加粗或以不同颜色/格式区分）
- [ ] 附录A的冲突对照表是否完整呈现了8个冲突项
