## 评审架构

本项目采用 **4 次评审、4 道关卡（Gate）** 的质量控制体系。每个 Phase 完成后进行一次整体评审，通过后方可进入下一阶段。

```
Phase-0 (0a+0b+0c)  ──→  [Gate 0 评审]  ──→  Phase-1 (1a+1b+1c+1d)
                                                        │
                                                [Gate 1 评审]
                                                        │
                                               Phase-2 (图表)
                                                        │
                                                [Gate 2 评审]
                                                        │
                                               Phase-3 (终版)
                                                        │
                                              [Gate 3 最终验收]
                                                        │
                                                    正式交付
```

## 评审流程说明

### 每次评审的标准流程

1. **准备**：收集该 Phase 的全部产出物，确认文件齐全
2. **硬性指标检查**：逐项核对 Gate 条件，记录通过/不通过
3. **质量评分**：按软性指标维度打分
4. **抽查验证**：对关键数据点进行抽样核对
5. **填写结论**：使用评审文件末尾的结论模板，记录判定结果
6. **决策**：
   - **通过** → 启动下一 Phase
   - **不通过** → 明确返工要求，返工完成后重新评审

### 评审纪律

- 评审结论必须以书面形式记录（填写结论模板），不接受口头通过
- 返工要求须具体到"哪个文件的哪个部分需要如何修改"
- 返工后的重新评审只需检查返工项，无需全量重审
- 评审结论保存在本文件夹内，作为项目过程记录

## 各 Gate 的关键关注点速查

| Gate | 核心问题 | 最高风险项 |
|------|---------|-----------|
| Gate 0 | 品牌情报是否覆盖充分？P&L计算是否正确？新内容是否已写入报告？ | P&L内部计算错误、品牌遗漏、整合遗漏 |
| Gate 1 | 审校是否全面？冲突决策是否明确？参考文献是否可靠？ | 高严重审校意见遗漏、冲突项无推荐方案、四份产出物内部矛盾 |
| Gate 2 | 图表数据是否准确？中文标注是否完整？视觉是否专业？ | 数据与报告不一致、中文乱码、分辨率不足 |
| Gate 3 | 终版是否落实了所有修改？执行摘要与终版是否一致？ | 审校意见未落实、执行摘要数据过时、图表显示异常 |
