# Phase-1 评审计划：审校与补充阶段整体评审

**评审范围**：Phase-1a（内容审校）+ Phase-1b（排版检查）+ Phase-1c（参考文献核实）+ Phase-1d（冲突决策）  
**评审时机**：Phase-1 四个子任务全部完成后、Phase-2 启动前（即 **Gate 1**）  
**评审对象文件**：

| 文件 | 来源 |
|------|------|
| `record/phase-1a_审校意见清单.md` | Phase-1a 产出 |
| `record/phase-1b_排版问题清单.md` | Phase-1b 产出 |
| `record/phase-1c_参考文献核实报告.md` | Phase-1c 产出 |
| `record/phase-1d_冲突决策备忘录.md` | Phase-1d 产出 |

**建议评审人**：项目负责人（Jz）  
**预计评审时长**：25–35 分钟

---

## 一、评审目标

整体验证 Phase-1 的四项审校任务是否充分发现了报告的问题，并为 Phase-2（图表制作）和 Phase-3（终版整合）提供了明确、可执行的修改清单和决策依据。

---

## 二、必须通过的硬性指标（Gate 1 条件）

### A. 内容审校（Phase-1a）

| # | 检查项 | 通过标准 | 结果 |
|---|--------|---------|------|
| A1 | 审校覆盖度 | 审校意见覆盖报告全部 12 个主章节 + 附录，无跳过的章节 | □ 通过 / □ 不通过 |
| A2 | 严重程度标注 | 每条审校意见标注了严重程度（高/中/低） | □ 通过 / □ 不通过 |
| A3 | 高严重遗漏 | 无"高严重"遗漏项被遗漏——即审校人需确认已检查了三份原始草稿中的核心数据点（如五年现金流模型、万锦Zoning细节、AODA无障碍法案等） | □ 通过 / □ 不通过 |
| A4 | 数字核对 | 至少对以下高敏感数据进行了核对：珍珠奶茶市场规模、盈亏平衡点、安省最低工资、HEYTEA定价 | □ 通过 / □ 不通过 |
| A5 | 产出格式 | 审校意见按5个分类（遗漏内容、数字错误、引用缺失、逻辑不一致、图表处理）结构化输出 | □ 通过 / □ 不通过 |

### B. 排版检查（Phase-1b）

| # | 检查项 | 通过标准 | 结果 |
|---|--------|---------|------|
| B1 | 检查完整性 | 覆盖全部 8 类检查项（页面设置、标题层级、目录、表格、分页、页眉页脚、参考文献格式、特殊标注） | □ 通过 / □ 不通过 |
| B2 | 表格逐一检查 | 报告中至少 9 张主要表格（含Phase-0新增表格）均已检查 | □ 通过 / □ 不通过 |
| B3 | 问题可操作性 | 每个排版问题都有"建议修复方式"，而非仅描述问题 | □ 通过 / □ 不通过 |

### C. 参考文献核实（Phase-1c）

| # | 检查项 | 通过标准 | 结果 |
|---|--------|---------|------|
| C1 | 核实覆盖率 | 第一优先级来源（Statistics Canada、CBRE、Health Canada等政府/权威机构）100%核实 | □ 通过 / □ 不通过 |
| C2 | 失效链接标记 | 所有 404/失效 URL 已明确列出，并给出处理建议（替换链接或移除） | □ 通过 / □ 不通过 |
| C3 | 汇总统计 | 报告开头包含汇总统计（总条数、可访问数、失效数、格式问题数） | □ 通过 / □ 不通过 |

### D. 冲突决策（Phase-1d）

| # | 检查项 | 通过标准 | 结果 |
|---|--------|---------|------|
| D1 | 8项冲突全覆盖 | 全部 8 个冲突项（投资总额、客单价、订单量、利润率、回本期、盈亏平衡、扩张时长、产品策略）均有决策分析 | □ 通过 / □ 不通过 |
| D2 | 推荐方案明确 | 每个冲突项给出了明确的推荐方案（非"待讨论"） | □ 通过 / □ 不通过 |
| D3 | 推荐有据 | 每个推荐方案附有至少 2 条论据支撑 | □ 通过 / □ 不通过 |
| D4 | 快速决策表 | 文末包含"快速决策一览表"，一目了然 | □ 通过 / □ 不通过 |

**判定规则**：A1–A5 + B1–B3 + C1–C3 + D1–D4 共 15 项全部通过 → Gate 1 放行。

## 三、质量评分维度（软性指标）

以下 6 个维度按 1–5 分评分，总分 ≥ 18 分（满分 30）为"良好"，≥ 24 分为"优秀"：

| 维度 | 评分标准 | 得分 |
|------|---------|------|
| **审校细致度** | 5=发现了10+条有价值的问题，含跨章节逻辑不一致等深层问题；3=发现5–10条基本问题；1=仅发现表面问题 | /5 |
| **排版问题实用性** | 5=问题描述精确到具体页/表/段，修复建议可直接执行；3=问题定位较笼统；1=仅泛泛描述"格式有问题" | /5 |
| **参考文献核实深度** | 5=不仅检查URL可访问性，还验证了标题/年份匹配度；3=仅检查URL可达；1=批量扫描未逐条验证 | /5 |
| **冲突分析质量** | 5=每个冲突分析了差异产生的根本原因（统计口径、业态假设、数据来源），论证有说服力；3=给出了推荐但分析较浅；1=仅重复数据未深入分析 | /5 |
| **冲突决策的保守偏好** | 5=投资相关冲突项的推荐明确偏保守，且解释了为什么保守更安全；3=中性取值；1=偏乐观 | /5 |
| **四份产出物的内部一致性** | 5=1a的审校意见与1d的冲突决策不矛盾，1c的失效链接在1a中也有提及；3=基本一致；1=存在矛盾 | /5 |

## 四、交叉一致性检查

Phase-1 的四份产出物之间应逻辑一致，评审人需检查以下关键交叉点：

- [ ] Phase-1a 发现的数字错误是否影响 Phase-1d 中冲突项的推荐值？如是，1d 是否已考虑？
- [ ] Phase-1c 发现的失效参考文献是否涉及 1a 中的高敏感数据点？如是，该数据点的可信度是否需要重新评估？
- [ ] Phase-1d 推荐的冲突决策值是否与 Phase-0b 的 P&L 模型假设一致？
- [ ] Phase-1b 发现的表格问题中，是否有因 Phase-0c 整合新增内容而导致的格式错乱？

---

## 五、对 Phase-2 的放行确认

Gate 1 通过后，启动 Phase-2 前需确认以下关键输入已就绪：

- [ ] Phase-1d 的"快速决策一览表"中的推荐值可直接用于图表制作
- [ ] Phase-1a 未发现影响图表数据准确性的"高严重"错误（如有，需先修正）
- [ ] 8 张图表所需的数据源（报告各章节+Phase-0b产出）中无已知的数据争议

---

## 六、常见返工原因

1. **审校覆盖不全**——跳过了某些章节（尤其新增的 2.5、4.1.1、8.0、8.4）
2. **冲突项缺少推荐**——仅列出差异但不给出明确推荐方案
3. **参考文献仅做表面检查**——只看URL是否返回200但未验证内容匹配度
4. **排版问题缺少修复建议**——Phase-3 的执行人无法据此修复
5. **四份产出物存在内部矛盾**——如1a说某数据有误但1d仍采用了该数据
